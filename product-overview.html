Australian Location Intelligence API

Geocoding, geofencing & real‑time location events — built on G‑NAF.

Wherabouts turns Australia's authoritative national address dataset into a clean REST API. Resolve addresses, draw zones, track devices, and get a webhook the instant something crosses a boundary — without standing up your own PostGIS, address‑matching, or event pipeline.

Core capabilities

One API for everything spatial in Australia

Every feature is exposed as a documented, key‑authenticated REST endpoint (Authorization: Bearer wh_live_…) with usage metering baked in.

📍

Forward & Reverse Geocoding

Turn a free‑form address into precise coordinates, or resolve a lat/lng back to the nearest real address. Country and state filters keep results tight.

  • GET /api/v1/addresses/geocode
⌨️

Address Autocomplete

Type‑ahead suggestions as a user types — minimum 2 characters, capped result sets — ideal for checkout and onboarding forms.

📦

Batch Geocoding

Submit thousands of addresses as an async job, poll for progress, then download results. No rate‑limit gymnastics for big datasets.

  • POST /api/v1/geocode/batch
  • GET /api/v1/geocode/batch/ { jobId}
  • GET /api/v1/geocode/batch/ { jobId}/results
🗺️

Zones & Geofencing

Create, update and delete polygon zones via API or the visual map editor. Group them per project, fit‑to‑view, and draw catchments in seconds.

  • POST/PUT/DELETE /api/v1/zones
🎯

Point‑in‑Zone & Addresses

Ask which zones contain a coordinate, or list every G‑NAF address that falls inside a zone — catchments, delivery areas, risk overlays.

  • GET /api/v1/zones/contains
  • GET /api/v1/zones/ { id}/addresses
🛰️

Device Location Tracking

Push a device's position and query its current zone membership. The platform maintains state so you don't have to diff coordinates yourself.

  • POST /api/v1/devices/ { id}/location
  • GET /api/v1/devices/ { id}/zones
🔔

Boundary‑Crossing Events

Every location update is diffed against prior membership to emit clean entry and exit events — the hard part of geofencing, solved.

🪝

Signed Webhooks

Subscribe to zone events and receive HMAC‑signed deliveries. Failing endpoints auto‑deactivate and can be reactivated — no polling required.

  • POST /api/v1/webhooks
  • POST /api/v1/webhooks/ { id}/reactivate
🔑

Keys, Projects & Usage

Multiple API keys, multi‑project workspaces, team members, a live API Explorer, and per‑key usage analytics with Server‑Timing headers.

Pain points solved

What you stop having to deal with

The recurring headaches of Australian location work — and how the platform removes each one.

The pain How Wherabouts solves it
Raw G‑NAF is hard to use. Authoritative but ships as bulk files needing ETL, normalisation and a spatial database. A managed REST API over G‑NAF — query addresses, coordinates and containment with a Bearer key, no data pipeline.
Bad addresses cost money. Typos drive failed deliveries, returns and checkout abandonment. Autocomplete + validation + geocoding ensure only real, resolvable addresses enter your system.
Geofencing logic is genuinely hard. Diffing positions to detect crossings without flapping is error‑prone. The platform tracks membership and emits clean, de‑duplicated entry / exit events for you.
Polling for location changes doesn't scale. Constant queries are wasteful and laggy. Signed webhooks push events as they happen; failing endpoints auto‑deactivate and reactivate.
Big datasets hit rate limits. Geocoding millions of rows synchronously is slow and brittle. Async batch jobs — submit, poll, download — handle scale without throttling errors.
Running spatial infra is overhead. PostGIS, tiling and address matching are a team's worth of work. Serverless API on Cloudflare Workers — zero infrastructure to operate or scale.
No visibility into spend or usage. Surprise bills and opaque consumption. Per‑key usage metering, analytics and clear tiers, with multi‑project and team separation.

API surface

The full endpoint map

All routes are versioned under /api/v1 and authenticated with a Bearer key.

Geocoding

  • GET /api/v1/addresses/geocode — forward geocode
  • POST /api/v1/geocode/batch — submit batch job
  • GET /api/v1/geocode/batch/ { jobId} — poll
  • GET /api/v1/geocode/batch/ { jobId}/results — download

Zones & Geofencing

  • POST /api/v1/zones — create
  • GET /api/v1/zones — list
  • GET /api/v1/zones/ { id} — get
  • PUT /api/v1/zones/ { id} — update
  • DELETE /api/v1/zones/ { id} — delete
  • GET /api/v1/zones/contains — point‑in‑zone
  • GET /api/v1/zones/ { id}/addresses — addresses in zone

Devices

  • POST /api/v1/devices/ { deviceId}/location — push location
  • GET /api/v1/devices/ { deviceId}/zones — current membership

Webhooks

  • POST /api/v1/webhooks — subscribe to zone events
  • GET /api/v1/webhooks — list subscriptions
  • DELETE /api/v1/webhooks/ { id} — delete
  • POST /api/v1/webhooks/ { id}/reactivate — reactivate

Endpoints, tiers and feature names in this document are drawn directly from the codebase (public API routers, the pricing route, and the docs page) as of this build.
