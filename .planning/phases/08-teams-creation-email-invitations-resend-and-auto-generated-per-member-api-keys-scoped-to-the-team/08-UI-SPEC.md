## Interaction Contracts

### Team page (`/team`)

**Members card**
- Render real member list from `useListOrganizations` / `useActiveOrganization`.
- Each row: Avatar (initials) + name + email + role Badge + optional Pending Badge + `···` menu (Remove member — owners/admins only, not on self).
- "Remove member" in `···` menu: opens an inline confirmation (`AlertDialog` or `window.confirm` is NOT acceptable — use `AlertDialog` from shadcn) with copy: "Remove [Name] from [Team Name]? Their API key will remain valid until used — it will be rejected by the API on next call." Two buttons: "Cancel" (ghost) and "Remove member" (destructive variant).

**Pending Invitations card**
- Render real invitation list from server fn.
- Each row: email + "Invited N days ago as [role]" + "Resend" (outline) + "Revoke" (ghost) buttons.
- Revoke: no confirmation dialog required (non-destructive to data; key is not generated until join).
- Empty state: hide the card entirely when no pending invitations exist (do not render an empty card).

**Invite Member dialog**
- Trigger: "Invite Member" button (top-right of page heading row).
- Fields: Email (`type="email"`, required) + Role select (options: "Member" selected by default, "Admin" — shown only to owners).
- Submit label: "Send invitation"
- On success: close dialog + toast "Invitation sent to [email]" + optimistic row in Pending Invitations.
- On error: inline error below email field: "Could not send invitation. Check the address and try again."
- Rate limit error (429): "Too many invitations sent. Wait a moment and try again."

### Invite acceptance route (`/invite/:id`)

**Unauthenticated visitor**
- Show centered card: team name + inviter name + "You've been invited to join [Team Name]".
- Single CTA: "Sign in to accept" (links to `/sign-in?redirect=/invite/:id`) or "Create account" below it.

**Authenticated visitor**
- Auto-call `acceptInvitation` on mount.
- On success: show one-time key modal (see below) then redirect to `/team`.
- On error (expired/already accepted): show card with "This invitation has expired or has already been used." + "Go to team" button.

**One-time key modal**
- Heading: "You're in — here's your API key"
- Body: "This is the only time you'll see this key in full. Copy it now." + monospace input showing plaintext key (read-only, full width).
- Actions: "Copy key" (primary, copies to clipboard + changes label to "Copied!" for 2s) + "I've copied it, continue" (ghost, dismisses and redirects to `/team`).
- Closing the modal via Escape or backdrop click is disabled (force-read pattern).

### Team switcher (sidebar)

- Hidden when user has only 1 team (Personal team, `teams.length < 2`).
- Visible: shows active team name as trigger button with chevron icon.
- Dropdown items: each team name (click → sets as active org) + `Separator` + "Create team…" item.
- "Create team…": opens an inline `Dialog` with a single text input for team name + "Create team" (primary) + "Cancel" (ghost).
- No confirmation on team switch — immediate.

### API key reveal (`/team` or `/api-keys`)

- Members see only their own key row with a "Reveal key" button (ghost, small).
- Clicking "Reveal key": calls `revealApiKeySecret` server fn → shows key in a modal (same one-time copy pattern as above, minus the "I've copied it, continue" redirect — just a "Done" button to close).
- Other members' rows: show only suffix (last 4 chars), name, last-used date, usage count — no reveal button.

## Email Template Contract (Resend)

Source: `08-CONTEXT.md` — "Exact shape of the Resend HTML email template" is Claude's discretion. Use project brand tokens.

- **Layout:** Single-column, max-width 600px, white background (`#ffffff`), dark text (`#1a1a1a`).
- **Font:** JetBrains Mono is a display/UI font — use system-safe stack for email: `ui-monospace, 'Courier New', monospace`. Email clients do not load Google Fonts reliably.
- **Structure:**
  1. Top bar: Wherabouts wordmark (text-only if no hosted logo asset).
  2. Heading (24px, semibold): "[Inviter Name] has invited you to join [Team Name] on Wherabouts"
  3. Body (14px, regular, line-height 1.5): "Click the button below to accept this invitation. This link expires in 72 hours."
  4. CTA button: background `#dedede` (matches `--primary` dark token), text `#1a1a1a`, border-radius 8px, padding 12px 24px, label "Accept invitation".
  5. Footer (12px, muted `#6b6b6b`): "If you weren't expecting this, you can ignore this email. This invitation was sent by [inviter email]."
- **Text fallback:** Plain-text version required — include team name, inviter name, and full invitation URL on its own line.
