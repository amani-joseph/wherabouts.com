## Task 1 — Pure confidence scorer

**Files:** `packages/api/src/shared/validation-confidence.ts` (create),
`packages/api/src/shared/validation-confidence.test.ts` (create)

1. `scoreValidation({ inputParts, candidate, candidateScore })` → `{ score: number (0-1),
   bucket: "verified"|"corrected"|"partial"|"unverified", corrections: Array<{field, from, to}> }`.
   Pure function — no db/env. Composite of three signals (document exact weights inline; flag the
   numeric tuning as an OPEN item from CONTEXT):
   - (a) **match strength**: the candidate's tier `similarity_score` (1.0 prefix / `similarity()`
     / `word_similarity()` / 0.5 levenshtein / 0.3 phonetic).
   - (b) **component agreement**: per-field comparison of input vs canonical
     (number_first, street_name+type+suffix, locality, state, postcode). Each mismatch that was
     auto-corrected lowers the score and appends to `corrections`.
   - (c) **canonical bonus**: small boost when the candidate carries a `gnaf_pid`.
2. Bucket thresholds: all components agree + high match → `verified`; corrections present but
   confident → `corrected`; missing/ambiguous components → `partial`; no usable candidate →
   `unverified` (score ~0).
3. Comprehensive unit tests: exact match → `verified` ~1.0; one misspelled street auto-corrected
   → `corrected` with a `corrections` entry; wrong postcode → downgrade; no candidate →
   `unverified` 0.

**Verify:** test suite covers each bucket boundary; function is deterministic and db-free.

**Commit:** `feat(api): pure G-NAF address validation confidence scorer`

## Task 3 — `/api/v1/addresses/validate` endpoint + tests

**Files:** `packages/api/src/routers/public/addresses-validate.ts` (create),
`packages/api/src/routers/public/index.ts` (edit — register),
`packages/api/src/routers/public/addresses-validate.test.ts` (create)

1. oRPC procedure modelled on `forwardGeocode`:
   - zod input: same discriminated union as geocode — `structured: "true"` with
     `street`/`locality` (+ optional `state`/`postcode`/`country`) OR `structured: "false"` with
     `q`. Reuse `buildGeocodeQuery` for the freeform path. Any numeric param → `z.coerce`.
   - `.use(apiKeyAuth)`, `.use(usageMiddleware("addresses.validate"))`.
   - OpenAPI `route`: `GET`, `path "/api/v1/addresses/validate"`, `tags: ["addresses"]`.
2. Flow: call `validateAddress` → pass `{ inputParts, candidate, candidateScore }` to
   `scoreValidation` → build `formattedAddress` from the canonical parts via
   `buildAddressLabel` (so it matches geocode output exactly).
3. Output shape: `{ standardizedAddress: { formattedAddress, streetAddress, locality, state,
   postcode, country, latitude, longitude, gnafPid }, confidence: number, status: bucket,
   corrections: [...] }`.
4. Register in `routers/public/index.ts`.
5. Tests mirror `geocode.test.ts`: structured vs freeform input both accepted; a corrected case
   returns `corrections` + `status: "corrected"`; an exact case returns `verified` high
   confidence; output `formattedAddress` matches what `buildAddressLabel` produces.

**Verify (criterion 3):**
- `GET /api/v1/addresses/validate?structured=false&q=1 martin place sydny` → corrected canonical
  address, `gnafPid` populated, `confidence` in (0,1], `status` populated.
- Exact valid address → `status: "verified"`, confidence near 1.0.
- `pnpm -F @wherabouts.com/api test` green.

**Commit:** `feat(api): GET /api/v1/addresses/validate with G-NAF confidence scoring`
