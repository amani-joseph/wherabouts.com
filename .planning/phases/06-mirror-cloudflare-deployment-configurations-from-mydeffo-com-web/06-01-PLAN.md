<objective>
Replace the Node.js filesystem-based env loader in packages/env/src/server.ts with the Worker-compatible dotenv/config pattern, and remove dead/unnecessary dependencies from apps/web/package.json.

Purpose: The current server.ts uses node:fs (existsSync) and node:path to walk directories for .env files. This will fail on Cloudflare Workers which have no filesystem. The dead dependencies (next, eslint-config-next, postcss tooling) add unnecessary bundle weight and potential conflicts.

Output: Worker-compatible server.ts, cleaned apps/web/package.json
</objective>

  <action>
Rewrite packages/env/src/server.ts to remove ALL node:fs and node:path usage. Replace the resolveEnvSearchRoots() and loadWorkspaceEnv() functions with a single `import "dotenv/config"` at the top.

The new file content should be:

```typescript
import "dotenv/config";
import { createEnv } from "@t3-oss/env-core";
import { z } from "zod";

export const serverEnv = createEnv({
  server: {
    DATABASE_URL: z.string().url(),
    BETTER_AUTH_SECRET: z.string().min(1),
    BETTER_AUTH_URL: z.string().url(),
    GITHUB_CLIENT_ID: z.string().min(1),
    GITHUB_CLIENT_SECRET: z.string().min(1),
    WEB_BASE_URL: z.string().url(),
    AUTH_COOKIE_DOMAIN: z.string().optional(),
    PORT: z.coerce.number().int().positive().default(3002),
  },
  runtimeEnv: {
    ...process.env,
    WEB_BASE_URL: process.env.WEB_BASE_URL ?? "http://localhost:3001",
  },
  emptyStringAsUndefined: true,
});
```

Key changes:
- Remove `import { existsSync } from "node:fs"` and `import path from "node:path"`
- Remove `resolveEnvSearchRoots()` function entirely
- Remove `loadWorkspaceEnv()` function entirely
- Remove `import { config as loadEnv } from "dotenv"` -- replaced by `import "dotenv/config"`
- Add `import "dotenv/config"` as first import (side-effect import that loads .env from cwd)
- Add `AUTH_COOKIE_DOMAIN: z.string().optional()` to the server schema (needed by Plan 03)
- Keep the WEB_BASE_URL fallback in runtimeEnv (preserves local dev behavior)

On Cloudflare Workers, `dotenv/config` is a harmless no-op (no .env file exists), and `nodejs_compat_populate_process_env` ensures wrangler vars/secrets populate process.env.
