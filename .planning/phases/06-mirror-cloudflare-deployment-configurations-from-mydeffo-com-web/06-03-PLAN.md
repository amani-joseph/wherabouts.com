Update packages/api/src/auth.ts to conditionally set the cookie `domain` attribute from `serverEnv.AUTH_COOKIE_DOMAIN`.

In the `advanced.defaultCookieAttributes` object, add a conditional `domain` property. The updated `advanced` block should be:

```typescript
advanced: {
  defaultCookieAttributes: {
    sameSite: "none",
    secure: true,
    httpOnly: true,
    ...(serverEnv.AUTH_COOKIE_DOMAIN
      ? { domain: serverEnv.AUTH_COOKIE_DOMAIN }
      : {}),
  },
},
```

This uses object spread to conditionally include `domain` only when AUTH_COOKIE_DOMAIN is set. When unset (local dev, workers.dev domains), no domain attribute is added and cookies behave as before (scoped to the exact origin). When set to `.wherabouts.com` in production, cookies are shared across all wherabouts.com subdomains.

Do NOT change any other part of the auth configuration. Keep trustedOrigins, emailAndPassword, database adapter, etc. exactly as they are.
