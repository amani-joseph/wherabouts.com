Modify 4 files in `packages/backend/convex/`:

**1. `convex.config.ts`** — Register the BetterAuth component. Replace existing contents with:

```typescript
import { defineApp } from "convex/server";
import betterAuth from "@convex-dev/better-auth/convex.config";

const app = defineApp();
app.use(betterAuth);
export default app;
```

**2. `auth.ts`** — Create BetterAuth client and auth factory. Create this NEW file with:

```typescript
import { betterAuth } from "better-auth/minimal";
import { createClient } from "@convex-dev/better-auth";
import type { DataModel } from "./_generated/dataModel";
import { components } from "./_generated/api";

export const authComponent = createClient<DataModel>(components.betterAuth);

export const createAuth = (ctx: unknown) => {
  return betterAuth({
    baseURL: process.env.VITE_CONVEX_SITE_URL,
    database: authComponent.adapter(ctx),
    emailAndPassword: {
      enabled: true,
    },
  });
};
```

Note: `baseURL` uses `VITE_CONVEX_SITE_URL` which is the `.convex.site` URL where HTTP actions are served. This env var must be set on the Convex deployment. The `emailAndPassword` plugin is enabled here for Phase 2 readiness.

**3. `auth.config.ts`** — Replace Clerk auth config with BetterAuth provider. Replace the entire file contents with:

```typescript
import { getAuthConfigProvider } from "@convex-dev/better-auth/auth-config";

export default {
  providers: [getAuthConfigProvider()],
};
```

This removes the `CLERK_JWT_ISSUER_DOMAIN` dependency and switches Convex's auth validation to use BetterAuth tokens.

**4. `http.ts`** — Create HTTP router with auth routes. Create this NEW file with:

```typescript
import { httpRouter } from "convex/server";
import { authComponent, createAuth } from "./auth";

const http = httpRouter();
authComponent.registerRoutes(http, createAuth);
export default http;
```

After creating all files, run `npx convex dev --once` from the `packages/backend` directory (or the appropriate convex command) to verify the Convex deployment accepts the new component and generates types. If `_generated` types need regeneration, this step handles it.
