## Task 2 — HTTP resilience layer (port of `http.ts`)

**Files:** `packages/sdk-python/src/wherabouts/_http.py`,
`packages/sdk-python/src/wherabouts/errors.py`

1. `errors.py` — `WheraboutsApiError(Exception)` with attributes `code`, `message`, `status`,
   `request_id`, `doc_url`, `fields` — a direct mirror of `packages/sdk/src/errors.ts`. Include
   the same code union as module constants for reference.
2. `_http.py` — a `Requester` built on `httpx.Client` mirroring `http.ts` exactly:
   - Constants: `DEFAULT_BASE_URL = "https://api.wherabouts.com"`, `DEFAULT_MAX_RETRIES = 2`,
     `DEFAULT_TIMEOUT = 30.0`, `BACKOFF_BASE = 0.2`, `BACKOFF_CAP = 5.0`,
     `RETRYABLE_STATUSES = {408,425,429,500,502,503,504}`, `WRITE_METHODS = {"POST","PUT"}`.
   - Base headers: `accept: application/json`, `authorization: Bearer <key>`,
     `x-wherabouts-sdk: py/<__version__> api/v1`.
   - Auto-generate an `Idempotency-Key` (`uuid4`) on writes and **reuse it across retries**.
     Honour a per-call `idempotency_key` override.
   - Retry loop: retry on `RETRYABLE_STATUSES` and on transport errors, full-jitter backoff
     (`random.uniform(0, min(cap, base * 2**attempt))`), honour `Retry-After` on `429` (parse
     integer seconds; sleep that long before the retry). Stop at `max_retries`.
   - Per-call overrides: `headers`, `max_retries`, `timeout`, `idempotency_key`.
   - On non-2xx: parse body `{ error: { code, message, request_id?, doc_url?, fields? } }`; read
     `request_id` from body, else from the `X-Request-Id` response header; raise
     `WheraboutsApiError`.
3. Keep `httpx` injectable (accept an optional `transport`/`client`) so tests can stub responses.

**Verify:** `mypy src` + `ruff check` clean; a smoke import works.

**Commit:** `feat(sdk-python): httpx resilience layer mirroring the TS http contract`

</task>
