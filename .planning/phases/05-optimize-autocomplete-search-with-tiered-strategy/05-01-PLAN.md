<objective>
Enable PostgreSQL extensions and add database columns/indexes needed for tiered autocomplete search.

Purpose: The current autocomplete uses only ILIKE with a trigram GIN index. Tiered search requires pg_trgm operators (similarity, word_similarity), fuzzystrmatch functions (levenshtein, dmetaphone), a B-tree index for fast prefix matching, and new columns for result ranking (population_score, admin_level).

Output: A SQL migration file and updated Drizzle schema ready for the query layer to consume.
</objective>

Create a hand-written SQL migration file `packages/database/drizzle/0009_tiered_search_extensions.sql` with these exact statements, separated by `--> statement-breakpoint` markers:

1. Enable extensions (idempotent):
```sql
CREATE EXTENSION IF NOT EXISTS pg_trgm;
--> statement-breakpoint
CREATE EXTENSION IF NOT EXISTS fuzzystrmatch;
```

2. Add new columns for ranking (with defaults so existing rows get values):
```sql
ALTER TABLE "addresses" ADD COLUMN IF NOT EXISTS "population_score" integer NOT NULL DEFAULT 0;
--> statement-breakpoint
ALTER TABLE "addresses" ADD COLUMN IF NOT EXISTS "admin_level" smallint NOT NULL DEFAULT 5;
```

`admin_level` values: 1=country, 2=state, 3=city/suburb, 4=street, 5=address (default). Lower number = higher admin importance for boosting.

`population_score` is 0 by default. Major cities will be backfilled separately (out of scope for this phase -- the column just needs to exist for the query layer to use).

3. Create B-tree index on search_text for prefix LIKE queries (the existing GIN trigram index handles similarity but B-tree is faster for `LIKE 'prefix%'`):
```sql
CREATE INDEX CONCURRENTLY IF NOT EXISTS "idx_addresses_search_text_btree"
ON "addresses" USING btree ("search_text" text_pattern_ops);
```

Note: `text_pattern_ops` enables efficient prefix matching with `LIKE 'foo%'` on a B-tree index.

4. Create index on population_score for ORDER BY ranking:
```sql
CREATE INDEX CONCURRENTLY IF NOT EXISTS "idx_addresses_population_score"
ON "addresses" ("population_score" DESC);
```

Do NOT drop or modify the existing `idx_addresses_search_text_trgm` GIN index -- it is still needed for trigram similarity queries.
