# Platform Phase 04: Intelligence, Scale and Enterprise

## Goal

Make Wherabouts defensible through differentiated intelligence, enterprise trust, and globally scalable platform behavior.

## Timeline

- Months 13-18

## Depends On

- Platform Phase 03

## Strategic Outcomes

- AI-powered address intelligence features
- enterprise controls and compliance posture
- global performance and scale improvements
- partner and developer-growth programs

## Strategic Requirements

- `P4-R01` Address intelligence differentiation
- `P4-R02` Enterprise trust posture
- `P4-R03` Global scale architecture
- `P4-R04` Growth leverage
- `P4-R05` Defensible platform strategy

## Milestones

- `M12` Intelligence layer
  - advances `P4-R01` and `P4-R05`
- `M13` Enterprise trust package
  - advances `P4-R02` and `P4-R05`
- `M14` Scale architecture
  - advances `P4-R03` and `P4-R05`
- `M15` Growth and ecosystem defensibility
  - advances `P4-R04` and `P4-R05`

## Candidate Tactical Workstreams

- add fuzzy matching and deduplication workflows
- define deliverability scoring model
- improve compliance, auditability, and residency options
- implement edge, caching, and load-testing strategy
- build partner, certification, and content-led growth programs

## Success Criteria

1. Wherabouts offers capabilities beyond commodity geocoding and lookup vendors.
2. Enterprise customers can evaluate the platform with confidence.
3. Global performance strategy is measurable and operationally meaningful.
4. Growth no longer depends only on direct product discovery.

## Repo Mapping

- API intelligence and scoring: `packages/api`, `packages/database`
- enterprise controls and observability: app/server/deployment layers
- partner and certification surfaces: docs, dashboard, and supporting web properties

## Readiness Gate

This phase should begin only after the platform has real product-market traction and observable platform usage patterns.

Differentiation work is most valuable once the core platform is already dependable.

## Prompting Reference

When requesting execution against this phase, reference both:

- milestone ID
- strategic requirement ID

Example prompt shape:

- "Plan `M12` in support of `P4-R01` and `P4-R05`."

## Open Questions

- Which intelligence feature delivers the strongest moat first?
- What compliance posture is necessary before private deployments are worth supporting?
- Which parts of the platform should be opened up to partners versus kept proprietary?
