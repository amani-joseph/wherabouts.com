Design §1 — Phase breakdown & build order

6 phases. Phase 0 is the shared foundation (sequential). Once it lands, phases 2–5 fan out in parallel worktrees. Decisions locked: MapLibre+MapTiler · build all backend gaps · add both history tables · hand-authored docs (GET-only explorer).

Phase 0 — Foundation (sequential, blocks everything)

0 · Map + oRPC + nav foundation

~biggest enabler · solo first
  • Install MapLibre GL + terra-draw; MapTiler key wiring (env)
  • Shared client-only <MapCanvas> (SSR-safe) — reused by zones + devices
  • oRPC convention: session protectedProcedure domains re-derive project ownership from session.user.id
  • Shared PostGIS helpers extracted so public + dashboard routers don't drift
  • Sidebar nav entries + route scaffolding for all new pages
▼ unlocks ▼
Phase 1 — Zones (depends on 0; defines map UX devices reuse)

1 · Zones Manager

map polygon draw/edit · keystone
  • oRPC zones.list/get/create/update/delete/contains/addresses
  • Map draw/edit polygons, zone list, PIP tester, addresses-in-zone viewer
  • 500-zone limit, ST_IsValid, closed-ring validation surfaced
⚠ devices (P4) reuses this map UX → P1 before P4
▼ then fan out in parallel ▼
Phases 2–5 — parallel (separate worktrees, mostly own files)

2 · Webhooks

forms/tables · no map
  • oRPC webhooks.list/create/delete/reactivate
  • + webhook_delivery_attempts table + log write-path in consumer
  • once-only secret reveal, failing badge + reactivate, delivery timeline
zone picker needs P1 zones.list (falls back to "all zones")

3 · Batch geocoding

forms/tables · no map
  • oRPC geocode.batchSubmit/Poll/Results/batchList
  • migration: batch_geocode_jobs.apiKeyId → nullable
  • paste/CSV input, polling progress, results table + CSV/JSON export, job history

4 · Device tracking

reuses P1 map
  • oRPC devices.list (new)/get/pushLocation
  • + device_location_history table + write-path in push handler
  • device list, map markers + zone overlays, trails, simulate-push tester
depends on P0 map + P1 zone overlays

5 · Docs / Explorer

hand-authored · GET-only try-it
  • 16 new endpoints documented across the 4 doc layers
  • try-it: GET executable; POST/PUT/DELETE = curl examples
  • async-lifecycle narrative (batch submit→poll→results; webhook delivery)
docs the real public API — independent of P1-4 UIs
