Which cluster should we prioritise first?

Pick the one that closes the highest-value gap for Wherabouts developers.

A

Forward Geocoding + Batch

Complete the geocoding surface — unstructured address text → lat/lng, plus async batch jobs. Rounds out the core API that developers expect before anything else.

B

Geofencing + Admin Boundaries

Custom polygon zones (CRUD + point-in-polygon) and AU administrative boundaries (suburb, LGA, state). High-value for logistics, delivery, and property use cases.

C

Location Context Layer

Enrich an address with its suburb/LGA, timezone, region codes, and statistical area. Makes every geocoded response carry more insight without needing a separate call.

D

All three — design them together

They're interconnected enough that designing the data model once (polygons + admin boundaries) enables both geofencing and context enrichment cleanly.
