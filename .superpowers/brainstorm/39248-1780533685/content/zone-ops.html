What can developers do with their zones?

Select all operations the geofencing API should support.

A

Create / Read / Update / Delete zones

Developers define named polygon zones via API (GeoJSON input). Store, list, fetch, update, and delete them. Scoped to their project.

B

Point-in-polygon check

Given a lat/lng, return which of the developer's zones contain that point. The core geofencing query — used for "is this driver inside my delivery zone?"

C

Addresses within a zone

Given a zone ID, return all addresses from the AU database that fall inside it. Useful for marketing lists, service area coverage, or bulk validation.

D

Entry / exit webhooks

Developers register a URL. When a tracked device crosses a zone boundary, Wherabouts fires a webhook. Requires a location update stream from the developer's app.
