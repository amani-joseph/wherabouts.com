Client polls GET /geocode/batch/ { jobId}. When done, response includes a pre-signed R2 URL valid for 1 hour. Max input size: 1,000 addresses per job (enforced at submit). Queue batch size: 10 per consumer invocation to stay within Worker CPU limits.

Edge Cases to resolve before planning

  • Zone limit per project: No unbounded polygon storage — recommend 500 zones/project on free tier, configurable per plan.
  • Concave / self-intersecting polygons: Reject on create with ST_IsValid(). Return 422 with specific error.
  • Device with no prior state: First location push has no previous zones — treat all containing zones as "entry" events immediately.
  • Webhook delivery failure: After 3 retries, mark subscription failing (not deleted). Developer can inspect and re-activate.
  • Addresses-in-zone for large zones: A zone covering all of NSW could match millions of rows. Cap at 10k results, return a truncated: true flag.
  • float32 → float64 migration: ALTER COLUMN on the live addresses table needs a migration window — low-risk in Postgres but worth flagging for ops.
